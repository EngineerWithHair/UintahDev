PSE Document Map

This document is a map to PSE documentation.

PSE documentation falls into the following categories:

PSE General Information Documents

These describe the concept of a PSE and the types of problems that the current set of PSEs solve.

PSE Administrator Documents

Documents for those who will be downloading and building the PSE.

PSE Network User Documents

These describe the purpose and use of predefined PSE networks.

PSE Network Builder Documents

Documents for those who construct networks for their own use and for use by others.

PSE Developer Documents

Documents for those who develop new modules, datatypes, converters, and other types of codes.

PSE Design Documents

Documents that describe the design of various pieces of the PSE.

General Information

  • SCIRun: A Scientific Programming Environment for Computational Steering (1995) describes the original PSE known as SCIRun.

  • The SCIRun Computational Steering Software System discusses the design, implementation, and application of SCIRun.

  • The SCI group has published many other papers.

  • Existing Problem Solving Environments (PSE):

    • SCIRun is a general purpose computational science problem solving environment developed by the Scientific Computing and Imaging Institute.

    • BioPSE is a problem solving environment for bioelectric field problems. It is developed by the NIH Center for Bioelectric Field Modeling, Simulation, and Visualization. BioPSE

    • Uintah is a problem solving environment for simulating fires involving a diverse range of accident scenarios including multiple high-energy devices, complex building/surroundings geometries and many fuel sources. It is developed by the Center for the Simulation of Accidental Fires and Explosions (C-SAFE).

    All PSE's allow interaction between the modeling, computation, and visualization phases of a simulation. All PSEs are extensible.

  • Subscribe to one or both of the mailing lists scirun-develop and scirun-interest. Do this by sending a message to majordomo@cs.utah.edu with the text "subscribe <name-of-list>" in the body of the message.

Developer

A developer writes modules, data converters, or other code. The following information is needed by developers.

  • PSE is implemented using C++. Here are a few C++ references:

    • Overview (with lots of links) of C++ by its original designer Bjarne Stroustrup.

    • "Highly Recommened" books for novice C++ programmers.

    • "Highly Recommened" books for advanced C++ programmers.

    • SGI's implementation of the Standard Template Library.

  • PSE graphical user interfaces are composed with the scripting language Tcl and its extensions Tk, iTcl, iTk, and BLT.

    There are a number of good books describing tcl/tk. One is Practical Programming in Tcl and Tk by Brent B. Welch.

    A new book explaining itcl (aka [incr Tcl/Tk]) is [incr Tcl/Tk] from the GROUND UP by Chad Smith.

  • Coding Standards have been developed in order to provide a basis for consistent, easily reviewable, and maintainable code. Your source code must be commented (documented) according to the PSE standard.

  • Browse PSE's source code (stuff from header files only).

  • PSE source code, documentation, and other files, are managed by the Concurrent Versions System software. Developers must be familiar with its use in the PSE project. Also see the cvs information prepared by the Uintah project.

  • New modules are created with PSE's built in " Module Maker " software.

  • Module writers must follow a specification procedure when writing new modules. Modules which are not fully spec'ed will not be incorporated into official PSE releases.

  • Data converters are standalone programs that convert "foreign" data to an existing PSE datatype object (PSE datatypes are "persistent" - data objects can be saved to a file and later recreated from the content of the file). You may need to write one some day.
