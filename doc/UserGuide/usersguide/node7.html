6. Importing Data Into SCIRun

The following illustrates some of the channels through which users can import data into SCIRun. The basic paradigm, at this point, requires converting existing files into standard SCIRun files. SCIRun will soon extend to upport for readers within SCIRun that accept other formats.

6.1 CVRTI Converters

The Cardiovascular Research and Training Institute (CVRTI) has developed a number of scalar and vector data and geometry file formats which require converters for SCIRun. The result is a set of stand-alone programs that read in one more more existing CVRTI file and generate a particular form of SCIRun file, the details of which depend on the particular combination of nodes, connectivities and associated attributes.

The programs and their parameters are as follows (go to the end of the list for definitions of the different parameter and file types):

CVRTItoTriSurfGrad:
converts nodes, triangle connectivities, and vector (grad) files into a field with associated vector attribute: 
        CVRTItoTriSurfGrad pts fac grad [channels] fieldout
CVRTItoTriSurfPot:
converts nodes, triangle connectivities, and scalar (pot) data files into a field with associated scalar attributes. 
        CVRTItoTriSurfPot pts fac pot [channels] fieldout
CVRTItoTetVolGrad:
converts nodes, tetrahedral connectivities, and vector (grad) files into a field with associated vector attributes: 
        CVRTItoTetVolGrad pts tetras grad [channels] fieldout
CVRTItoTetVolPot:
converts nodes, tetrahedral connectivities, and scalar (pot) files into a field with associated scalar attributes: 
        CVRTItoTetVolPot pts tetras pot [channels] fieldout

6.1.0.1 Definitions:

Argument Purpose
Geometry
pts CVRTI points file; ASCII file with one x,y,z triplet per line.*
fac CVRTI triangle connectivity (facet) file; ASCII file with one triangle defined per line. Each value points to a node number in the associated .pts file, with 1 (not 0) as the first node number.*
tetras CVRTI tetrahedra file; ASCII file with the nodes from one tetrahedron on each line, pointing to the nodes in the associated .pts file. Pointers begin with 1 (not 0).*
Data
grad CVRTI vector file; ASCII file with two x,y,z triplets per line; first triplet is origin of the vector and second is the endpoint.*
pot CVRTI scalar (potentials) data file; ASCII file, each line contains one scalar value. Without .channels file, all programs assume a one-to-one mapping of scalar value to the nodes in the associated .pts file.*
Channel Mapping
channels CVRTI data channel mapping file; ASCII file that begins with the line ``N channels'', where N is the number of channels in the file. Subsequent lines contain two values, the first refers to a node number in the geometry and the second points to the associated channel of any scalar or vector data files. Thus a .channels file must have an entry for each node of the associated .pts file but the associated .pot file can have more (or even fewer) entries.
Field File
field SCIRun fields file that contains both the geometry and associated scalar or vector data as attributes.
  *Note: to find the length of all CVRTI ASCII geometry, scalar, and vector files, count the number of lines in the file.

6.1.0.2 Location of Converter Programs:

All converter programs are normally available in the subdirectory of the SCIRun distribution called src/StandAlone/convert. If they are not easily available, ask the local SCIRun installer for assistance.
