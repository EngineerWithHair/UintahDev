5. Visualization with the Viewer

 

This section describes, perhaps, the most frequently used module of SCIRun, the Viewer, which displays interactive graphical output to the computer screen. Use the Viewer to see a geometry, or spatial data. The Viewer provides access to many simulation parameters and controls, thus, indirectly initiates new iterations of the simulation steps important to computational steering (described in Computational Steering).

 

   
5.1 Anatomy of the Viewer Window

 

The Viewer window contains two main areas: the upper portion, Graphics window which displays the graphics, and the lower portion where most of the control buttons are located. Figure 5 contains an example of a Viewer window. In the Graphics window, the mouse, mouse buttons, and various modifier keys (shift/control/alt) determine control. The lower window contains several buttons and sliders, the function of which becomes clear upon reading this guide.


  
Figure 5: The default Viewer window in SCIRun

First, control the Graphics window by moving the mouse to the center of the viewer window, click and hold the left button, dragging the mouse. The objects should translate accodingly. Perform the same operation with the middle mouse button and the objects will rotate around a point in the center of the display. The right mouse button controls the scale of the display, zooming in when the mouse moves downward or to the right. See Mouse Control in the Viewer for details about mouse control.

Basic configurations may be set by using the control along the bottom of the Viewer window:

restores the display back to a default condition. This is very useful when objects disappear from the view window due to a combination of settings.
captures the setting of the current view so you can return to it later by clicking the "Go home'' button.
restores the current home view.

From the Viewer window, the left corner of the control panel contains performance indicators that document the current rendering speed for the display. More advanced graphics performance results in a higher drawing rate.

A small plus sign (``+'') appears in the lower right corner of the Viewer window. Clicking on the plus signreveals the extended control panel with controls described in Extended control window.

5.1.1 Menus

Two pull-down menus appear at the top of the Viewer window.

allows selection from among graphics hardware settings that are available on each workstation. The list is ordered heuristically from most-to-least useful.

5.3 Extended Control Window

 

Click on the ``+'' sign in the lower right corner of the default Viewer window, and the window expands to reveal an extended panel of control buttons, as shown in Figure 6. Click on the "-'' sign that now replaces the ``+'' and this extended panel disappears again. The following table describes the control options available in the extended control window:


  
Figure 6: The lower portion of extended Viewer window in SCIRun

5.3.1 Object Selector

The lower portion of the extended Viewer window is divided into three columns.The middle column contains a list of all the objects in the display. If the list becomes long enough, a scroll bar on the left controls which are visible. Each entry in the list has the following controls, reading from left to right:

  • A red box appears to the left of each selected object. The Viewer window only displays those objects that are selected.
  • Next comes the name of the object.
  • The control box determines the shading options used for rendering the object. Options include: Lighting, BBox, Fog, Use Clip, Back Cull, and Display List (for descriptions, see Rendering controls).
  • The lighting control appears to the right of each entry, initially marked . In the Default setting, the common rendering controls, described in Rendering controls below, apply. Clicking this box reveals a set of options that will apply only to this object that include Wire, Flat and Gouraud.

   
5.3.2 Rendering Controls

 

The left column of the extended Viewer window contains controls that apply to all of the selected objects with ``Default'' lighting selected. Those without the Default setting will use their own, object-specific settings, as described in the previous section. The lighting and shading options available are:

Lighting:
toggles whether or not the Viewer applies lighting to the display. Objects without lighting have a constant color.
Fog:
draws objects with variable intensity based on their distance from the user, also known as ``depth cueing.'' As a function of distance from the front, close objects appear brighter while more remote objects fade gradually into the background
BBox:
toggles whether the Viewer draws the selected objects in full detail or as a simple bounding box.
Use Clip:
applies up to six clipping planes to the display. To control the clipping plane locations, use the ``Edit -> Clipping Planes'' menu at the top.
Back Cull:
displays only the forward facing facets of any surface objects in the display.
Display List:
cache the list of objects to be displayed; this option accelerates rendering when the content of the display does not change.
Shading:
selects the type of shading for objects from the following options:
Wire:
show only the wire mesh of objects.
Flat:
draw each facet with a constant color.
Gouraud:
interpolate linearity of the color across facets.

The right hand column of the extended Viewer window contains controls for displaying the axes and creating stereoscopic rendering.

5.3.2.1 Stereo Viewing:

requires hardware LCD glasses synchronized with the display so that visibility for each eye coincides with the display of the appropriate view. The ``Fusion Scale'' control provides a means of setting the eye separation, thus, setting the view that is most suited to facial anatomy and distance from the screen.

     
5.3.3 Making Movies

The Viewer window in SCIRun has simple controls for capturing sequences of images into animations or movies.

Controls in the left column of the extended Viewer window select movie type and then initiate or stop the acquisition of individual frames in the movie.

SCIRun sends a frame to the movie after each "redraw'' operation, ( i.e., each time anything moves in the display or any visualization parameter changes). If the MPEG package is available (see the Installation Manual for details) then saving the animations as MPEG movies will be an option.

There is also a button that forces the size of the graphical window to be 352x240 pixels in size, a standard format well suited to MPEG.

   
5.4 Control Widgets

While the mouse controls describe many ways to interact with the contents of the Viewer, SCIRun also supports some powerful display widgets. Examples of widgets' capabilities include: manage cutting surfaces colored according to the local data values; display streamlines in vector fields; or select sub-volumes within the display area for further manipulation. Widget interaction is consistent so that, for example, clicking and dragging on either a cylindrical "collar'' or a sphere element controls parameters. James Purcifal generated the original design of these modules. Note that a single widget may have more than one purpose, depending on the context in which it exists.

In this section, we describe the widgets available within SCIRun and BioPSE. The same widget may, for example, select a clipping or a display plane through a three-dimensional object but may also set the seed points for a streamline module.

   
5.4.1 Gauge Widget


  
Figure 7: The gauge widget for setting location and density of seed points

5.4.1.1 Appearance:

The Gauge Widget consists of two spheres (A) connected by a cylinder (B) with a small slider collar (C) on the cylinder. There are also small resize cylinders extending from the spheres (D).

5.4.1.2 Purpose:

The primary use of the Gauge Widget is to set the location and density of streamlines emerging from the long cylinder. It may also be used as a more general purpose three-dimensional slider or as a source for a stream surface.

5.4.1.3 Controls:

Clicking and dragging either sphere causes the entire widget to move in space, rotating about the other sphere and following along behind the selected sphere. Dragging either of the resize cylinders causes the size of the widget to change. Dragging any point on the main cylinder moves the whole widget without any change in orientation. Dragging the slider collar changes the associated value, typically, the density of seed points for a streamline source.

   
5.4.2 Frame Widget


  
Figure 8: The frame widget for selecting cutting/projection planes

5.4.2.1 Appearance:

The Frame Widget consists of four cylinders connected in a rectangle. In the middle of each of the cylinders there is a sphere (B), from which a resize cylinder extends (C).

5.4.2.2 Purpose:

The primary uses of the Frame Widget is for image plane definition, for defining stream volumes, and as a "tie dye" as with the Ring Widget described in Ring Widget.

5.4.2.3 Controls:

Clicking and ragging a sphere on the widget will cause the widget to rotate about it center; dragging on a resize cylinder will move the associated edge and this extend or contract the rectangle. Dragging any of the cylinder will drag the entire widget through space.

   
5.4.3 Box Widget


  
Figure 9: The boxwidget for selecting sub-volumes

5.4.3.1 Appearance:

The Box Widget consists of twelve cylinders (A) connected in a hexahedral box (three-dimensional rectangle) with cylinders indicated on the edges of the box (B). A sphere with a cylinder protruding from it appears in the middle of each face of the box, (C) providing resize control.

5.4.3.2 Purpose:

The primary use of the Box Widget is to select a subvolume of the workspace for further manipulation (e.g., volume rendering, isosurfaces, streamlines, mesh adaption) where the faces of the widget act as orthogonal clipping planes.

5.4.3.3 Controls:

Clicking and dragging on one of the spheres rotates the widget about its center without changing the position of the center. Clicking on and dragging any resize handle causes the associated face to extend without changing its orientation. Dragging a cylinder causes the entire widget to move without changing its orientation.

   
5.4.4 Ring Widget


  
Figure 10: The ring widget for selecting cutting/projection planes

5.4.4.1 Appearance:

The Ring Widget consists of a ring (A) with four embedded spheres (B), each with a resize cylindrical attached (D). A sliding collar (C) exists between two of the spheres. One of the resize cylinders has a special material property (typically a different color from the other cylinders) to indicate that it is the ``halfway point'' for the slider (E).

5.4.4.2 Purpose:

The primary use of the Ring Widget is to set the density of streamlines emerging from the ring. The ring serves as a set of seed points from which emerge streamlines. The Ring Widget can also serve as a three-dimensional angle gauge as a source for multiple streamlines throughout its surface; as a source for a stream surface from the outer ring and as a source for a stream volume. A color sheet, or "tie-dye" provides abnother use as a function of the scalar value of the field at each point.

5.4.4.3 Controls:

Clicking and dragging the slider collar along the ring changes the density of the seed points or some other related parameter. Dragging the spheres controls the orientation of the Ring Widget, while moving the resize cylinders changes the radius of the Ring Widget about its center. Dragging any other point on the ring moves the ring in space without changing its radius or orientation.
