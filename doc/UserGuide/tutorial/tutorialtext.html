Introduction: Using this Tutorial

By simultaneously running SCIRun and this tutorial, the user can easily follow these instructions. This tutorial also serves as a visual reference guide toward building a sample net. To demonstrate this process, users are introduced to the SCIRun software and to the BioPSE package through a forward finite element problem.

This tutorial is not meant as a comprehensive guide to the SCIRun/BioPSE software system. You will benefit greatly by reading the full documentation contents for SCIRun /BioPSE. Additionally, for more descriptive information about tutorial modules, you may access the module catalog.

You can also click on the various statements that are linked throughout this document. If your browser supports frames, visual cues will appear on the right window as a guide to building your net.

 

Problem Statement

Using a volume conductor model and an equivalent dipole source, we hope to compute the potentials and electric field induced by the source through the domain. An analytic solution exists for the problem of very specific geometric models (e.g., analytic cylinders or spheres). However, for a real-world problem, only discretization determines the solution. For this problem, we discretize our domain into tetrahedral finite elements where each element contains a conductivity tensor. This defines how electricity travels through its region of the domain. Within each element, we will assume a piece-wise linear potential field (piece-wise constant electric field).

Getting Started

Computing the Electric and Potential Fields

Computing the electric and potential fields given a finite element volume conductor mesh, and a dipolar current source, we follow these steps:

1)Read in the finite element mesh (tet elements, with a conductivity tensors for each cell) [mesh]

2) Read in a set of dipole sources [source]
3) Construct the finite element stiffness matrix based on the conductivity mesh [A]
4) Construct the right hand side source vector based on the location and orientation of any dipole sources [b]
5) Solve the linear system , to determine the potentials at the nodes []
6) Attach the solution vector, , to the mesh in order to give the potentials a spatial location [potential field]
7) Take the gradient of the potential field () to get the electric field []

For Steps 1 and 2, we use FieldReader modules to read in the mesh and a dipole. To access the FieldReader module, first select SCIRun>Data IO>FieldReader from the menu bar. The FieldReader module appears in the upper left corner of the network pane. Note that various modules contain a user interface (UI) button. Clicking these give you a full view of the module's interface, as well as the ability to change variables at any point in your computation. For this tutorial, we use the sphere dataset. You can also run [it] on the cyl3 dataset. Our first FieldReader reads in the mesh which, in this case, is a sphere.

** Note that SCIRun can be installed into any directory, within a user home directory or a system application directory. There is no "make install" step in the build process, so SCIRun will be installed in the chosen directory.

The chosen install directory will be referred to as DATA_DIR from now on.

For example, if you choose to install SCIRun into user "Bob's" home directory, then

DATA_DIR = /home/bob

In the user interface, set the filename to be: DATA_DIR/sphere-mesh.tvt.fld.

You can move the module by left-clicking on the module and dragging it to your desired position. Releasing your finger will set the module in place.

For Step 3, we use a SetupFEMatrix module to construct the linear operator A from the mesh. To achieve this, select BioPSE>Forward>SetupFEMatrix from the menu bar. Once the module appears in the network pane, you can drag it beneath the existing FieldReader module.

You can leave all of the default settings which are:

  • Select the "No Dirichlet" option so you do not specify any information regarding the boundary conditions.
  • Select "Use Conductivities," assuming this is a nonhomogeneous volume conductor.

For Step 4, we use an ApplyFEMCurrentSource module to build the righthand side source vector b, based on the location and orientation of the dipoles relative to the mesh. Select: BioPSE>Forward>ApplyFEMCurrentSource

You should position this module to the right of the SetupFEMatrix.

In the user interface, select the "source model" option to be "dipole." With this option we do not specify a sink and source electrode.

For Step 5, we use a SolveMatrix module to solve for the potentials, , in the linear system, . Select SCIRun>Math>SolveMatrix from the menu bar. Position the module beneath the SetupFEMatrix in the network pane.

Now we can set the SolveMatrix variables.

  • The Solution Method we want is "Conjugate Gradient & Precond. (SCI)"
  • Set the Target error to be 0.0001
  • Set the Maximum Iterations to be 200.
  • Uncheck "Use previous solution as initial guess"
  • Set the Precondition to Jacobi

* NOTE: the SolveMatrix window must either be open or minimized, and must not be closed completely.

Now that you have the first four modules positioned in the network pane, you should begin connecting them (though, you can connect or disconnect modules at any time). The first connection, referred to as a data pipe, goes between the FieldReader and the SetupFEMatrix. Note that the FieldReader contains only one output port. To connect the modules, center-click and hold down on the output port of the FieldReader. When you center-click on the output port of any module, a series of black lines run between all viable connections. As you roll the cursor over the lines, they will highlight in red. Releasing on the highlighted line will complete this piped connection in yellow. For our purposes, you should connect to the first input port on the SetupFEMatrix. This process applies to all inter-module connections.

The next connections will be between:

  • The FieldReader module output port to the first ApplyFEMCurrentSource module input port.
  • The first output port of the SetupFEMatrix module to the first input port of the SolveMatrix module.
  • The first output port of the ApplyFEMCurrentSource module to the second input port of the SolveMatrix module.


For Step 6, we use a ManageFieldData module (SCIRun>Fields>ManageFieldData) to attach the data values (potentials) to the geometry (finite element mesh) in order to build a potential field. This will be connected to the first input port of the Gradient module (Step 7).

For Step 7, we use a Gradient module (SCIRun>Fields>Gradient) to compute the electric field (), from the potential field. This will eventually connect to the first input port of the SeedField module.


Having solved for the electric and potential fields for a static source, there are two interactions we can now explore: A) moving the dipole within the model; B) visualizing the results of our computations.


Moving the Dipoles

To move the dipoles, we insert a ShowDipoles module (BioPSE>Visualization>ShowDipoles) between a new dipole source FieldReader module (filename: DATA_DIR/sphere-dipole.pcv.fld) and the ApplyFEMCurrentSource module.

By connecting the Viewer module, (SCIRun>Render>Viewer), to the ShowDipoles module, we can begin to visualize the location and orientation of the source, and move it around in the model by selecting its representative scene graph widget.

In the user interface, set the option to "Scale Size" and scale the widget to 0.05. Check the option to "Show Last As Vector."

To execute the network, right click on any module. This will bring up a menu with various options. Select "Execute."

Visualizing the Electric and Potential Fields

To visualize the electric field (vectors) and the potential field (scalars), we have a number of visualization modules with which you can experiment. First, we'll examine an isopotential surface.

We can attach a data pipe to feed the potential field from the ManageFieldData module into an Isosurface module (SCIRun>Visualization>Isosurface). Our isosurface potentials can be mapped to a range of data values. To do this, we must construct an appropriate colormap.

We add a GenStandardColorMaps module, (SCIRun>Visualization>GenStandardColorMaps), to the canvas and select an attractive colormap from its user interface. In order to scale that colormap so it corresponds to the range of datavalues in our potential field, we feed (connect) the colormap and the potential field into a RescaleColorMap module (SCIRun>Visualization> RescaleColorMap). The output of RescaleColorMap is a colormap with min/max values mapped to the corresponding values from the potential field. This new colormap, along with the potential field, can then be passed (connected) into an Isosurface module.

Isosurface variables:

  • Adjust the Iso Value slider to be 0.1000
  • Under the Options section, set "Auto Extract from the New Field"
  • Leave the Method as using "MC" which is Marching Cubes and the number of processors, "np," to be 1.

By changing the isovalue on the Isosurface interface, we can visualize any isopotential surface within the model.

In addition to isosurfacing, we can also view potential values at specific locations within the field. We can interpolate or extrapolate potentials at arbitrary locations using the BuildInterpolant and ApplyInterpolant modules. For example, we read-in a collection of electrode positions (represented as a PointCloud) with a new FieldReader module (DATA_DIR/sphere-electrodes.pcd.fld), feed the geometric mesh and the electrode positions to the BuildInterpolant module (SCIRun>Fields>BuildInterpolant).

The resulting output is an interpolant field, indicating the weights and indices from the mesh required to compute the data values at the electrode locations. The interpolant field and the potential field can then be fed into an ApplyInterpolant module (SCIRun>Fields>ApplyInterpolant) to compute those interpolated values (multiplying the kernels by the source data values to get the destination data values).

We can then visualize the results, using a ShowField module (SCIRun>Visualization>ShowField). To see the electrodes color-mapped, we feed the rescaled colormap into the ShowField module along with the potential-mapped electrode field.

ShowField variables:

  • Show Nodes as Spheres with a Node Scale of 0.03 (all defaults)
  • Show Edges as Lines with a Cylinder Scale of 0.03 (defaults)
  • Show the Faces (defaults)
  • Set the Cylinder and Sphere Resolution to be 4 (default) and leave the default color.


Lastly, we can visualize the electric field by tracking streamlines through the domain (SCIRun>Visualization>StreamLines). These streamlines are the electric field lines set up by the dipole sources.

StreamLine variables:

  • Error Tolerance to be 0.01
  • Set the Step Size to be 0.1
  • Set the Maximum Steps to be 200

To choose the seed positions for the streamlines, we use a SeedField module (SCIRun>Fields>SeedField). SeedField generates a collection of seed points based on either a scene widget (e.g. a "rake") or by randomly distributing them over the input domain. The electric field and the seed positions pass into the Streamline module; seed positions integrate through the (gradient) vector field. Variable settings can be left at default. The output from the Streamline module is a collection of Streamlines (represented as contours), which we can then visualize by, again, using a ShowField module.

Variables for the second ShowField module should be:

  • Show Nodes as Spheres with a Node Scale of 0.03 (all defaults)
  • Do not show Edges
  • Do not show Faces
  • Set the Cylinder and Sphere Resolution to be 10 and leave the default color.

You have now built the forward-fem.net using SCIRun. From an individual module's interface, you can adjust settings and re-execute the net. To visualize the results, try different Iso values or move the StreamLines rake by pressing Ctrl-left-mouse and dragging the rake to the desired position. You can do the same with the dipole source.

For this tutorial we used a simple sphere geometry from which an analytic solution can be obtained. This same network can also be applied to more realistic models such as a head or torso mesh in which discretization is necessary.
